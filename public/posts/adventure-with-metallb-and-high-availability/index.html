Adventure With MetalLB and High availability

January 21, 2025

I am going to attempt to make my home lab Highly Available. I only have a master node and an agent node at the moment. More machines can be added in the future in the short term I intend to leverage MetalLB with ARP (Address Resolution Protocol)) mode in conjunction with multiple nodes that handle the same IP address.

In ARP Mode MetalLB will assign the same virtual IP to multiple nodes in the cluster. Only one node will respond to ARP at a time. If that node goes down the other node will take over. This node can be either the master or the agent node.

I followed the following steps;

  1. Install MetalLB
helm repo add metallb https://metallb.github.io/metallb && \
helm repo update
  1. Create namespace for the deployment
kubectl create namespace metallb && \
kubectl config set-context --current --namespace metallb
  1. Deploy the chart
helm install metallb metallb/metallb
  1. Create IP Address Pool Resource
apiVersion: metallb.io/v1beta1
kind: IPAddressPool
metadata:
  name: default-pool
  namespace: metallb
spec:
   addresses:
   - 192.168.1.200-192.168.1.255

The above means that the IP addressess that will be assigned will be from 192.168.1.200 to 192.168.1.255. Make sure that this does not clash with your DHCP server!

  1. Create Layer2 Advertisement Resource
apiVersion: metallb.io/v1beta1
kind: L2Advertisement
metadata:
  name: default-pool-advertisement
  namespace: metallb
spec:
  ipAddressPools:
  - default-pool
  1. Apply the above resources
kubectl apply -f ip-address-pool.yaml
kubectl apply -f l2-advertisement.yaml

Take time in doing these steps. Watch the logs to ensure everything is up and happy.
