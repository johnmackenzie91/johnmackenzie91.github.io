Setting Up Longhorn with Block Storage on My HA K3s Cluster

February 17, 2025

Introduction
As part of my journey to build a high-availability (HA) k3s cluster, I decided to use Longhorn for persistent storage. While exploring the best storage configuration, I learned about the advantages of using block storage over filesystem storage. Here‚Äôs a detailed guide on how I prepared my disk for Longhorn and configured it to use raw block storage for maximum performance.

Why Block Storage?#

After some research, I found that Longhorn supports two types of storage: filesystem and block. Here‚Äôs why I chose block storage:

  • üíæ Efficient Space Utilization: No file system means Longhorn fully manages the storage.

Checking My Disk Type#

I inspected my node‚Äôs storage configuration with:

lsblk -f

I found /dev/sda was formatted with ext4, so I needed to wipe it for block storage.

Wiping the Disk for Block Storage#

I prepared the disk for block storage with the following commands:

sudo umount /dev/sda || true
sudo wipefs --all /dev/sda
sudo dd if=/dev/zero of=/dev/sda bs=1M count=100 oflag=direct,dsync
sudo blkdiscard /dev/sda || true
sudo parted /dev/sda --script mklabel gpt
lsblk -f /dev/sda

The disk now showed no FSTYPE, confirming it was a raw block device.

Format Disk and Mount Disk#

Format and Mount the Disk (if needed):

sudo mkfs.ext4 /dev/sda

sudo mkdir -p /mnt/longhorn/disk1

Mount the disk

sudo mount /dev/sda /mnt/longhorn/disk1

Ensure it mounts on reboot: Add to /etc/fstab:

echo "/dev/sda /mnt/longhorn/disk1 ext4 defaults 0 2" | sudo tee -a /etc/fstab

Ensure fstab is configured correctly with;

sudo mount -a

Configuring Longhorn to Use the Block Disk#

Option A: Longhorn UI#

In the Longhorn UI, I added the disk under Node and Disk Management with:

  • Path: /dev/sda
  • Storage Type: block
  • Allow Scheduling: true

Option B: Helm Values File#

I updated values.yaml:

longhorn:
  nodes:
    - name: node-master-1
      disks:
        sda:
          path: /dev/sda
          allowScheduling: true
          storageReserved: 0
          tags:
            - fast-storage

Then deployed with Helm:

helm upgrade --install longhorn longhorn/longhorn \
  --namespace longhorn-system \
  -f values.yaml

Final Verification#

I verified everything was working with:

kubectl get nodes.longhorn.io -n longhorn-system
kubectl get disks.longhorn.io -n longhorn-system

Everything looked great!

Conclusion#

By preparing /dev/sda as a raw block device and auto-mounting it without a filesystem via /etc/fstab, I ensured my HA k3s cluster has a reliable, high-performance storage backend. Longhorn with block storage offers excellent performance and scalability. If you‚Äôre setting up Longhorn, I highly recommend using block storage for dedicated disks.
